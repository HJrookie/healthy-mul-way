## 本次（SSG 预渲染 + 语言写进 URL）实现

用户核心诉求：原 hash 路由 + 纯 CSR 导致百度读不到正文、sitemap 的 `#/article` 链接不友好。改为 vite-ssg 在构建时给每篇文章预渲染独立 HTML（自带 title/meta，非 JS 注入），并把语言写进 URL（`/zh`、`/en`），配合 Netlify 解决收录问题。用户声明已自行部署 Netlify（https://healthy.24992345.xyz），故只本地构建 + present_files，**不公开部署**。

### 关键技术决策（重要坑）
- **依赖甄别**：用户口述的 `vite-plugin-ssg` 实为 React 专用包（peer 需要 react）。正确 Vue SSG 包是 **`vite-ssg`**（antfu，v28.3.0），搭配 `vue-router@^4` + `@unhead/vue`，构建命令 `vite-ssg build`。
- **@unhead/vue 版本坑（关键）**：`vite-ssg` 将 `@unhead/vue@^2` 作为自身依赖内置；若项目顶层装 `@unhead/vue@3`，会出现「两个版本」——vite-ssg 用 v2 创建 head 实例，组件用 v3 的 `useHead` 注入，注入键不兼容 → head 完全不生效（首页/文章页都回落到默认 title）。**必须将 `@unhead/vue` 锁到 `^2.1.2`**，让 npm 去重成单一副本（已锁 2.1.15）。
- **文章路由写法**：`routes.js` 中文章路由路径写成字面量 `/${lang}/article/${a.id}`（每条静态路由），文章 id 通过 `meta.articleId` 传给 ArticleView（**不是** `:id` 动态段，否则 `route.params.id` 为 undefined）。
- **base 与目录式输出**：`vite.config.js` 改 `base: '/'`（history 模式要求绝对基路径）。vite-ssg 默认输出扁平 `.html`（`dist/zh.html`），为兼容所有静态主机，新增 `scripts/fix-dist-paths.mjs` 在 build 后把扁平文件转成目录式 `dist/zh/index.html`、`dist/zh/article/<id>/index.html`。
- **Netlify**：`netlify.toml` 设置 build=`npm run build`、publish=`dist`，并重定向 `/* → /index.html 200`（SPA 兜底；已预渲染的静态文件优先于 splat 命中）。
